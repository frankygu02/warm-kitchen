# 暖味厨房家庭版上线说明

这套版本不再使用 Puter，也不要求家人登录任何 AI 平台。家人只需打开网页、输入共同的“家庭访问码”，网站就会通过安全的云端中转调用 DeepSeek。

DeepSeek 密钥和家庭访问码都只能填写在 EdgeOne 的环境变量中，不能写进网页、GitHub 文件或聊天消息。

## 第一次上线

1. 打开腾讯云 EdgeOne Pages（Pages 也称“边缘 Pages”），选择“从 GitHub 导入项目”。
2. 授权并选择仓库 `frankygu02/warm-kitchen`，生产分支选择 `main`。
3. 这是纯网页项目：框架选择“其他”或“静态 HTML”，不需要构建命令，输出目录选择项目根目录。
4. 在项目的“设置 → 环境变量”中新增：

   - `DEEPSEEK_API_KEY`：您的 DeepSeek API 密钥。
   - `FAMILY_ACCESS_CODE`：您自己设置的家庭访问码，建议至少 8 位，不要与银行卡或常用密码相同。
   - `DAILY_FAMILY_LIMIT`：建议填写 `60`，表示全家每天最多生成 60 次。
   - `DAILY_DEVICE_LIMIT`：建议填写 `20`，表示每台手机或电脑每天最多生成 20 次。
   - `DEEPSEEK_MODEL`：填写 `deepseek-v4-flash`。

5. 不需要申请或绑定 KV。网站使用免费版可用的 Makers Blob 保存每日次数；第一次调用推荐接口时，平台会自动创建名为 `warm-kitchen-limits` 的存储空间。
6. 重新部署项目。完成后，先打开 EdgeOne 网址后面的 `/api/recommend`；看到 `configured: true` 和 `counterStorage: "makers-blob"` 即表示云端设置正确。
7. 返回网站首页，输入食材和家庭访问码测试一次。第一次请求后，可在项目的 Blob 存储页面看到自动创建的存储空间和计数记录。

## 分享给家人

只需把 EdgeOne 提供的网址和家庭访问码分别发给家人。建议网址与访问码分成两条消息发送。家人的设备会记住访问码，以后通常无需重复输入。

网页仓库即使是公开的，也看不到 DeepSeek 密钥和家庭访问码；真正的保护来自 EdgeOne 环境变量、家庭访问码和每日次数限制。待 EdgeOne 网站运行稳定后，可以再把 GitHub 仓库改为私有，并确认 EdgeOne 仍有访问该私有仓库的权限。

## 日后修改

- 想更换家庭访问码：只改 EdgeOne 中的 `FAMILY_ACCESS_CODE`，然后重新部署。
- 怀疑密钥泄露：到 DeepSeek 平台删除旧密钥并创建新密钥，再更新 `DEEPSEEK_API_KEY`。
- 想控制费用：调低 `DAILY_FAMILY_LIMIT` 和 `DAILY_DEVICE_LIMIT`。即使云端 AI 暂时不可用，网站仍会自动显示本地菜谱。
